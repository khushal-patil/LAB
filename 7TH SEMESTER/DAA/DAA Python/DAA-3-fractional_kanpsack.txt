## 5) Viva: most asked questions + short answers (memorize these)

1. **What is fractional knapsack?**
   Pick full or fractional items to maximize value under capacity.

2. **Greedy choice here?**
   Take item with maximum value/weight ratio first.

3. **Why greedy works here but not for 0/1 knapsack?**
   Fractional choices make the local optimum extendable to global; 0/1 has combinatorial constraints.

4. **Input / output of program?**
   Input: n, weights[], values[], capacity. Output: maximum achievable value (float).

5. **What does ratio mean?**
   Value per unit weight â€” benefit density.

6. **When do you take a fraction?**
   When remaining capacity < current item weight.

7. **Time complexity?**
   O(n log n).

8. **Space complexity?**
   O(n).

9. **Edge cases to handle?**
   Zero capacity, zero items, item with weight 0 (handle separately), negative values (usually invalid).

10. **How to prove optimality?**
    Exchange argument: replacing any lower-ratio part with higher-ratio part increases or keeps total value.

11. **What if items have equal ratio?**
    Any order among them yields same result.

12. **Real-world uses?**
    Resource allocation, cargo loading, fractional resource leasing.

13. **Difference b/w fractional and 0/1?**
    Fractional allows splitting; 0/1 does not â€” 0/1 requires DP for exact optimum.

14. **Can greedy be modified for integer knapsack?**
    No â€” greedy may fail; use dynamic programming or branch-and-bound.

15. **What data structure to use if items stream in?**
    Maintain a max-heap on ratio; extract as needed.

16. **Handling precision?**
    Use floats/doubles; print formatted to 2 decimals in practicals.

17. **If values and weights are large integers?**
    Use double precision or big integers for intermediate products if needed; but ratio is float.

18. **How to measure time fairly?**
    Use `time.perf_counter()` and average over multiple runs.

19. **What happens if total weights â‰¤ capacity?**
    Take all items â€” result = sum of all values.

20. **Why sort by ratio and not by value or weight alone?**
    Ratio captures benefit per unit weight â€” the right heuristic for capacity-constrained profit.

21. **Is greedy always fastest?**
    It's fast and optimal for fractional variant; complexity dominated by sorting.

22. **How to return selected fractions?**
    Record fraction taken per item (1.0 for full, <1.0 for partial) while iterating.

23. **What if item weight is zero and value>0?**
    The ratio is infinite â€” pick that first; handle separately to avoid division by zero.

24. **How to adapt to integer weights only?**
    Still fractional if you allow fractions; else use 0/1 knapsack DP.

25. **Can you compute an upper bound for 0/1 knapsack using fractional knapsack?**
    Yes â€” fractional solution gives an upper bound used in branch-and-bound.

26. **What is the greedy invariant?**
    After choosing k highest-ratio parts, the remaining capacity filled optimally by remaining items.

27. **What metric to print in practical?**
    Print max value, fractions chosen, and time taken.

28. **Implementation pitfalls?**
    Wrong sort key (use value/weight), integer division in some languages, division by zero.

29. **Why value/weight (not weight/value)?**
    We maximize value for limited weight; ratio must measure value per unit weight.

30. **How to demonstrate in viva?**
    Show sorted list by ratio, then stepwise filling, and final value (use n=3 example above).

## 2) Explain Knapsack with example using greedy approach (worked example)

**Problem:** fractional knapsack â€” you may take fractions of items. Goal: maximize total value subject to capacity.
**Greedy rule:** pick items in descending order of **value/weight** (value per unit weight).

**Choose capacity = 50 (typical practical example).**

Step-by-step (sorted by ratio: 1 â†’ 2 â†’ 5 â†’ 4 â†’ 3):

1. Take **Item 1** fully (weight 5 â‰¤ 50).

   * Capacity left = 50 âˆ’ 5 = **45**.
   * Value taken = **30**.

2. Take **Item 2** fully (weight 10 â‰¤ 45).

   * Capacity left = 45 âˆ’ 10 = **35**.
   * Value taken = 30 + **40** = **70**.

3. Take **Item 5** fully (weight 25 â‰¤ 35).

   * Capacity left = 35 âˆ’ 25 = **10**.
   * Value taken = 70 + **90** = **160**.

4. Next best is Item 4 (ratio 3.5) but weight 22 > remaining capacity 10 â†’ take **fraction** = 10/22.

   * Value added = 77 * (10/22) = 77 * 0.454545... = **35.00** (exact: 3.5 * 10 = 35).
   * Capacity left = 0.
   * Final total value = 160 + 35 = **195.00**.

5. Item 3 not taken (no remaining capacity).

**Result:**

* Selected: Item1 (full), Item2 (full), Item5 (full), Item4 (10/22 fraction).
* **Maximum value = 195.00** for capacity 50.
