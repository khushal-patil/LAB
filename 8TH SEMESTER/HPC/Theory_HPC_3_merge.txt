# 📌 4. MERGE FUNCTION (VERY IMPORTANT)

```cpp
void merge(vector<int>& arr, int left, int mid, int right)
```

### 🔹 Purpose:

👉 Combine two sorted arrays into one

---

## 🔹 Step 1: Create temp arrays

```cpp
vector<int> L(n1), R(n2);
```

---

## 🔹 Step 2: Copy data

```cpp
L[i] = arr[left + i];
R[i] = arr[mid + 1 + i];
```

---

## 🔹 Step 3: Merge

```cpp
arr[k++] = (L[i] <= R[j]) ? L[i++] : R[j++];
```

👉 Compare and insert smaller element

# 📌 6. PARALLEL MERGE SORT (MOST IMPORTANT)

---

## 🔹 Helper Function

```cpp
parallelMergeSortHelper(arr, left, right, depth)
```

👉 `depth` controls number of threads

---

## 🔹 Key Idea

```cpp
if (depth <= 0)
```

👉 Stop parallelism → switch to sequential

---

## 🔹 TASK CREATION

```cpp
#pragma omp task
parallelMergeSortHelper(left half)
```

```cpp
#pragma omp task
parallelMergeSortHelper(right half)
```

👉 Each half runs in **parallel**

---

## 🔴 TASKWAIT (VERY IMPORTANT)

```cpp
#pragma omp taskwait
```

👉 Wait until both tasks complete

---

## 🔹 Merge Step

```cpp
merge(arr, left, mid, right);
```

# 📌 7. MAIN PARALLEL REGION

```cpp
#pragma omp parallel
{
    #pragma omp single
    parallelMergeSortHelper(...)
}
```

### 🔹 Explain this clearly:

| Directive  | Meaning                      |
| ---------- | ---------------------------- |
| `parallel` | Create threads               |
| `single`   | Only one thread starts tasks |

👉 Other threads execute tasks

# 📌 10. ALGORITHM (WRITE IN EXAM)

---

### 🔹 Sequential Merge Sort

1. Divide array
2. Recursively sort halves
3. Merge sorted halves

---

### 🔹 Parallel Merge Sort

1. Divide array
2. Create parallel tasks for both halves
3. Wait for tasks to complete
4. Merge results

# 📌 12. VIVA QUESTIONS (VERY IMPORTANT)

---

## 🔹 Q1: Why merge sort is better than bubble sort?

👉 Because:

* Time complexity:

```text
O(n log n)
```

---

## 🔹 Q2: What is divide and conquer?

👉 Breaking problem into smaller subproblems

---

## 🔹 Q3: What is OpenMP task?

👉 A unit of work executed by threads

---

## 🔹 Q4: Why `#pragma omp task` used?

👉 To run recursive calls in parallel

---

## 🔹 Q5: What is `taskwait`?

👉 Wait for all tasks to complete

---

## 🔹 Q6: Why `single` used?

👉 To avoid multiple threads creating duplicate tasks

---

## 🔹 Q7: What is depth parameter?

👉 Controls recursion parallelism

👉 Prevents:

* Too many threads
* Overhead

---

## 🔹 Q8: Time complexity?

👉

```text
O(n log n)
```

---

## 🔹 Q9: Space complexity?

👉

```text
O(n)
```

👉 Due to temporary arrays

---

## 🔹 Q10: Why merge sort is parallel-friendly?

👉 Independent subproblems

---

## 🔹 Q11: What happens without taskwait?

👉 Incorrect merging (race condition)

---

## 🔹 Q12: What is scalability?

👉 Performance improves with more cores

---

## 🔹 Q13: Why use OpenMP instead of threads?

👉 Easier:

* Less code
* Automatic thread management

---

## 🔹 Q14: Difference: parallel bubble vs merge?

| Feature     | Bubble  | Merge      |
| ----------- | ------- | ---------- |
| Complexity  | O(n²)   | O(n log n) |
| Parallelism | Limited | High       |
| Efficiency  | Low     | High       |

---
