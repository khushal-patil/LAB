# 📌 2. THEORY (What you must say)

### 🔹 What is Bubble Sort?

👉 Repeatedly compares adjacent elements and swaps them.

### 🔹 Key Idea:

* After each pass → largest element goes to end

---

### 🔹 Parallel Bubble Sort (IMPORTANT)

👉 Normal bubble sort is sequential
👉 So we use:

### ✅ **Odd-Even Transposition Sort**

👉 Works in two phases:

1. Odd phase → compare (1,2), (3,4)
2. Even phase → compare (0,1), (2,3)

👉 This allows **parallel execution**

# 📌 4. SEQUENTIAL BUBBLE SORT

```cpp
void sequentialBubbleSort(vector<int>& arr)
```

### 🔹 Logic:

```cpp
for (int i = 0; i < n - 1; i++)
    for (int j = 0; j < n - i - 1; j++)
```

👉 Compare adjacent elements:

```cpp
if (arr[j] > arr[j + 1])
    swap(arr[j], arr[j + 1]);
```

---

### 🔹 Time Complexity:

👉 Worst case:

```text
O(n²)
```

---

# 📌 5. PARALLEL BUBBLE SORT (IMPORTANT)

```cpp
void parallelBubbleSort(vector<int>& arr)
```

---

## 🔹 Outer Loop

```cpp
for (int i = 0; i < n; i++)
```

👉 Each iteration = one phase (odd/even)

---

## 🔹 Parallel Loop

```cpp
#pragma omp parallel for
```

👉 Splits loop among threads

---

## 🔹 Key Logic

```cpp
for (int j = i % 2; j < n - 1; j += 2)
```

👉 THIS IS VERY IMPORTANT 👇

| i value | Phase      |
| ------- | ---------- |
| even i  | even phase |
| odd i   | odd phase  |

---

### 🔹 Example:

Array:

```
5 3 8 2
```

Even phase:

```
(0,1), (2,3)
```

Odd phase:

```
(1,2)
```

---

## 🔹 Swap

```cpp
if (arr[j] > arr[j + 1])
    swap(arr[j], arr[j + 1]);
```

👉 Independent swaps → safe for parallel

# 📌 8. ALGORITHM (WRITE IN EXAM)

---

### 🔹 Sequential Bubble Sort

1. Compare adjacent elements
2. Swap if needed
3. Repeat for all passes

---

### 🔹 Parallel Bubble Sort

1. Repeat for n phases
2. If even phase:

   * Compare (0,1), (2,3)
3. If odd phase:

   * Compare (1,2), (3,4)
4. Perform comparisons in parallel

# 📌 9. VIVA QUESTIONS (VERY IMPORTANT)

---

## 🔹 Q1: Why bubble sort is not directly parallel?

👉 Because:

* Each swap depends on previous swap

---

## 🔹 Q2: How did you parallelize it?

👉 Using:

> Odd-Even Transposition Method

---

## 🔹 Q3: What is OpenMP?

👉 API for parallel programming (shared memory)

---

## 🔹 Q4: What does `#pragma omp parallel for` do?

👉 Divides loop among threads

---

## 🔹 Q5: What is speedup?

👉

```text
Speedup = Sequential Time / Parallel Time
```

---

## 🔹 Q6: Ideal speedup?

👉 Equal to number of cores

---

## 🔹 Q7: Why sometimes speedup < 1?

👉 Due to:

* Thread overhead
* Synchronization cost

---

## 🔹 Q8: Time complexity of parallel bubble sort?

👉 Still:

```text
O(n²)
```

👉 But faster due to parallel execution

---

## 🔹 Q9: Why odd-even method works?

👉 Because:

* No overlapping comparisons
* Safe for parallel threads

---

## 🔹 Q10: What is scalability?

👉 Ability to improve performance with more cores

---

## 🔹 Q11: Difference: Bubble vs Merge Sort?

| Bubble Sort      | Merge Sort      |
| ---------------- | --------------- |
| O(n²)            | O(n log n)      |
| Easy             | Complex         |
| Poor parallelism | Highly parallel |

---

## 🔹 Q12: Why merge sort is better for parallel?

👉 Because:

* Divide and conquer
* Independent subproblems

---

## 🔹 Q13: What happens without `parallel for`?

👉 Code becomes sequential

---

## 🔹 Q14: Why no critical section here?

👉 Because:

* Each thread works on independent indices

---

## 🔹 Q15: What is load balancing?

👉 Equal work distribution among threads
